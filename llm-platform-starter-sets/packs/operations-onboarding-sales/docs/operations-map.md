# Operations Map

This document describes the core operational workflows for TNDS client delivery.
It is a retrieval source for the ops-playbook module.

---

## The Direction Protocol (Sales)

Five-stage consultative sales process.

```
Identify (15-20 min)
    → Is there a fit?

Assess (30 min)
    → Is the problem real?

Map (90-120 min)
    → What's broken, what's needed?

Chart (30-45 min)
    → Present findings and quote

Launch
    → Close and kickoff
```

**Key phrase:** "Direction Protocol gets you clarity. Command Protocol gets you control."

---

## The Command Protocol (Delivery)

Three service tiers.

### Command Center Build
- **Price:** $2,500 – $5,000
- **Timeline:** 2–4 weeks
- **What it is:** Install visibility + Command modules. Fixed scope, fixed price.

### Battle Rhythm Install
- **Price:** $2,000 – $4,000
- **Timeline:** 2–4 weeks
- **What it is:** Install operating cadence + accountability systems.

### Command Partner
- **Price:** $1,000 – $2,000/month
- **What it is:** Ongoing fractional ops support.

## Client Onboarding Flow

```
Contract signed
    ↓
Kickoff call scheduled
    ↓
Workspace setup (folder structure, permissions, labels)
    ↓
Data import or connection
    ↓
Module install and configuration
    ↓
Training session
    ↓
First battle rhythm check-in (Week 2)
    ↓
30-day review
```

---

## Delivery Standards

All deliverables must be:
- Production-ready (not proof-of-concept)
- Error-handled
- Input-validated
- Documented for non-technical clients

**No open-ended projects. No surprise invoices.**
